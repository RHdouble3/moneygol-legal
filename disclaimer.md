# 면책 조항 (내부용)

> **이 문서의 목적:** MoneyGoal 앱 서비스 운영 중 발생할 수 있는 분쟁 상황에서
> 책임 범위를 사전에 명확히 정의하는 내부 기준 문서.
> 이용약관(공개용) 작성 시 "면책 조항" 섹션의 근거로 사용한다.
>
> **마지막 업데이트:** 2026-05-13
> **다음 검토 시점:** 새 기능 추가 / 외부 서비스 연동 / 출시 직전

---

## 1. 앱의 정체성 (가장 중요한 전제)

MoneyGoal은 **사용자가 자신의 매매 기록을 직접 입력하고 관리하는 일지(diary) 앱**이다.

다음 어디에도 해당하지 않는다:

| 우리 앱이 아닌 것 | 이유 |
|----------------|------|
| ❌ 투자자문업 | 자본시장법상 면허 필요, 우리는 면허 없음 + 자문 행위 안 함 |
| ❌ 투자권유 서비스 | "이 종목 사세요" / "이 가격에 파세요" 같은 권유 일체 없음 |
| ❌ 시세 정보 제공 서비스 | 외부에서 실시간 시세·뉴스 가져오지 않음 (확인됨: Finnhub은 종목명 검색 전용) |
| ❌ 자동 매매 시스템 | 실제 매매 주문 기능 없음 |
| ❌ 수익 보장 서비스 | "수익 보장", "원금 보장" 같은 표현 일체 없음 |

> **앱의 정체성 코드 레벨 확인 (2026-05-13):**
> - 추천 매매원칙 화면 (`principle_list_page.dart:553-567`): "추천 원칙을 선택하거나 직접 추가해보세요" — 권유 문구 없음
> - 온보딩 (`onboarding_page.dart:25-36`): "기록 앱", "통계", "원칙" 강조, 수익 약속 없음
> - 외부 API 호출: Finnhub `/search` 엔드포인트만 사용, 시세/뉴스 미수신

## 2. 면책 사항 목록

### 2-1. 투자 결과에 대한 면책

- 사용자가 본 앱에 기록한 매매 결과(수익·손실)는 **사용자 본인의 매매 결정의 결과**이다.
- 본 앱은 매매 결정에 영향을 주는 어떠한 조언·추천·예측도 제공하지 않는다.
- 사용자가 본 앱의 통계·차트·원칙 준수율 등을 참고하여 향후 매매를 결정했더라도,
  그 결과로 발생한 손익은 **전적으로 사용자 본인의 책임**이다.

### 2-2. "추천 매매원칙"에 대한 면책

- 본 앱이 회원가입 시 제공하는 추천 매매원칙(예: 손절선 -5%, 5일선 이탈 시 매도 등)은
  **일반적으로 알려진 매매 규칙의 예시**이다.
- 이는 사용자가 자신만의 매매원칙을 쉽게 시작할 수 있도록 제공되는 **참고 예시**이며,
  본 앱이 해당 원칙의 효과성·수익성을 보장하거나 권유하는 것이 아니다.
- 사용자는 추천 원칙을 사용하지 않을 수 있으며, 자신만의 원칙을 자유롭게 작성할 수 있다.
- **추천 원칙을 적용한 매매 결과에 대한 책임은 사용자 본인에게 있다.**

### 2-3. 통계 및 차트 정확성에 대한 면책

- 본 앱의 모든 통계(월별 손익, 승률, 누적 손익, 원칙 준수율 등)는
  **사용자가 직접 입력한 데이터를 기반으로 계산**된다.
- 사용자가 잘못된 정보(매수가, 매도가, 수량, 매매일 등)를 입력한 경우
  통계 결과 또한 부정확해지며, 이로 인한 결과는 사용자 책임이다.
- 본 앱은 사용자가 입력한 데이터의 정확성을 검증하지 않는다.

### 2-4. 종목 검색 결과에 대한 면책

- 종목 검색 시 표시되는 종목명·종목코드는 외부 데이터(KRX, Finnhub)에 기반한다.
- 외부 데이터의 정확성·최신성·완전성에 대해 본 앱은 보장하지 않는다.
- 표시되지 않는 종목, 잘못 표시된 종목 등으로 인한 매매 기록 오류는 사용자 책임이다.

### 2-5. 서비스 중단·데이터 손실에 대한 면책

다음 사유로 인한 서비스 중단·지연·데이터 손실에 대해 본 앱은 책임지지 않는다:

- **외부 서비스 장애** — Supabase(데이터 저장), Finnhub(종목 검색 API), 인증 제공자(구글·애플 등)의 장애·점검·정책 변경
- **천재지변·전쟁·테러** 등 불가항력
- **사용자 기기 문제** — 기기 분실·고장·OS 업그레이드 실패 등
- **사용자의 앱 삭제·재설치** — 기기 로컬 저장 데이터(시드/목표/테마/온보딩 상태) 손실
- **회원 탈퇴** — 회원 탈퇴 시 모든 데이터가 즉시 완전 삭제되며 복구되지 않음 (Free 플랜, 백업 없음)
- **네트워크 장애** — 사용자 측 인터넷 연결 문제로 인한 데이터 동기화 실패

### 2-6. 비밀번호 분실·계정 도용에 대한 면책

- 비밀번호는 Supabase Auth가 해시 처리하므로 본 앱 운영자도 알 수 없다.
- 비밀번호 분실 시 이메일을 통한 재설정 외에 다른 복구 방법이 없다.
- 사용자가 비밀번호를 타인에게 알려주거나 부주의로 노출하여 계정이 도용된 경우,
  이로 인한 데이터 변경·삭제 등의 결과는 사용자 책임이다.

### 2-7. 만 14세 미만 가입에 대한 면책

- 본 앱은 **만 14세 이상**만 가입 가능하며, 회원가입 시 동의를 통해 확인한다.
- 사용자가 연령을 허위로 입력하여 가입한 경우, 발생하는 모든 문제는 사용자(및 보호자) 책임이다.

### 2-8. 출시 후 추가될 기능에 대한 면책

다음 기능은 출시 후 추가될 수 있으며, 추가 시점에 본 면책 조항도 갱신된다:

- 소셜 로그인 (구글/애플/카카오/네이버) — 외부 인증 제공자 정책에 따른 면책 추가
- 푸시 알림 — 알림 미도착·중복 발송 등에 대한 면책 추가
- 커뮤니티 기능 — 사용자 게시물에 대한 면책 추가
- 결제·구독 기능 — 결제 처리에 대한 면책 추가

## 3. 면책되지 않는 부분 (운영자 책임)

면책 조항이 광범위해도 **다음은 운영자의 책임**으로 남는다:

- **고의·중대한 과실로 인한 개인정보 유출**
- **개인정보처리방침을 위반한 데이터 수집·이용**
- **법령상 의무 위반** (예: 만 14세 미만 가입 허용 등)
- **운영자가 코드로 작성한 버그로 인한 명백한 데이터 손상** (단, 사용자 입력 실수는 제외)

> 한국 법령상 약관에서 사업자의 **고의·중대한 과실**을 면책시키는 조항은 무효이다.
> 따라서 면책 조항은 "통상의 운영 범위" 내에서만 적용된다.

---

## 4. 분쟁 처리

- 본 앱 이용 중 발생한 분쟁은 우선 **앱 내 문의 또는 운영자 이메일을 통해 해결**한다.
- 합의가 되지 않을 경우 한국 법령 및 관할 법원의 판단에 따른다.
- 운영자 연락처는 출시 시점에 추가 (이메일·문의 채널)

---

## 5. 면책 조항의 효력

- 본 면책 조항은 **이용약관의 일부**로서 사용자가 회원가입 시 동의한 것으로 본다.
- 면책 조항 일부가 법령에 의해 무효가 되어도 나머지 조항은 효력을 유지한다.
- 면책 조항 갱신 시 앱 내 공지 또는 이메일로 알린다.

## 6. 공개용 이용약관 반영 시 주의사항

본 내부 문서를 이용약관에 반영할 때:

- [ ] "투자 권유 아님" 명시는 **눈에 띄게** 강조 (붉은 글씨, 굵은 글씨 등)
- [ ] "수익 보장 안 함" 표현 명시
- [ ] 회원 탈퇴 시 **"복구 불가능"** 표현 명시 (Free 플랜)
- [ ] 추천 매매원칙은 "예시일 뿐"임을 명시
- [ ] 외부 서비스 장애 면책 — Supabase, Finnhub 이름 직접 노출 여부 결정 필요
- [ ] 면책 조항이 너무 길면 사용자가 안 읽음 → **핵심 5개 정도로 압축한 요약본** 별도 표시 검토
